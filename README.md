## Why ClaudeStudio?

The Claude Code CLI is extraordinarily powerful, but it lives in a terminal ‚ÄĒ and it forgets. ClaudeStudio fixes both.

- **No hidden knowledge required.** Worktrees, hooks, skills, MCP servers, `AGENTS.md`, plan mode, permissions ‚ÄĒ every advanced Claude Code capability is surfaced through a discoverable, native UI. You don't have to memorize flags or hand-edit config to use the good parts.
- **Permanent semantic memory.** Conversations, decisions, and artifacts are archived and embedded (Qdrant + SQLite) so Claude can recall what happened last week ‚ÄĒ or last month ‚ÄĒ instead of starting cold every session.
- **A real Agentic OS, not a wrapper.** A Supervisor, an Event-Bus, and agent-to-agent (A2A) messaging let multiple agents collaborate on long-running work. ClaudeStudio is built as an operating layer for agents, not a chat box bolted onto a binary.

## ‚ú® Feature Highlights

#### ūüßĎ‚ÄćūüöÄ Agent Studio & Teams
Compose, configure, and save individual agents ‚ÄĒ then assemble them into **Teams** that tackle a goal together. Each agent carries its own role, tools, permissions, and memory scope.

#### ūüß† Agentic OS (Supervisor ¬∑ Event-Bus ¬∑ A2A)
A **Supervisor** plans and delegates, an **Event-Bus** fans work out and collects results, and **agent-to-agent (A2A)** messaging lets agents hand off, request help, and negotiate ‚ÄĒ the backbone for durable, multi-step automation.

#### ūüóāÔłŹ Permanent Session Archive
Every session is captured and replayable. Nothing is lost when a window closes ‚ÄĒ browse, search, and resume past work with full context.

#### ūüĒé Semantic Memory (Qdrant + SQLite)
Vector search over your history (Qdrant) plus structured metadata and relationships (SQLite). Claude retrieves the *relevant* past, not just the recent past.

#### ūüēłÔłŹ Brain View ‚ÄĒ Knowledge Graph
A live, navigable graph of entities, files, decisions, and their relationships across all your sessions. See *how* your project's knowledge connects, not just *what* was said.

#### ūüéôÔłŹ Voice Assistant
Talk to your agents. Dictate tasks, get spoken status, and drive workflows hands-free.

#### ūüďö Definition Library
Reusable building blocks ‚ÄĒ agent definitions, skills, hooks, and prompts ‚ÄĒ versioned and shareable so your best setups become first-class assets.

#### ‚ö° Task Library ‚ÄĒ One-Click Workflows

#### ūüĒĆ MCP & Hooks Managers
Discover, install, configure, and toggle **MCP servers** and lifecycle **hooks** from a UI ‚ÄĒ no manual JSON surgery, with validation and live status.

#### ūüíį Cost & Telemetry

#### ūüõ°ÔłŹ Trust Modes & Permissions

## ūüŹóÔłŹ Architecture

ClaudeStudio is a **native SwiftUI front-end** talking to a **Rust core sidecar** over a local Unix domain socket using MessagePack framing. The Rust core owns all the heavy lifting ‚ÄĒ orchestration, memory, persistence, and process management ‚ÄĒ and drives the Claude Code CLI.

```mermaid
flowchart TB
    subgraph mac["macOS Application"]
        UI["SwiftUI Front-End<br/>Agent Studio ¬∑ Brain View ¬∑ Voice"]
    end

    UI <-->|"Unix domain socket<br/>(MessagePack RPC + events)"| CORE

    subgraph rust["Rust Core Sidecar"]
        CORE["Supervisor ¬∑ Event-Bus ¬∑ A2A<br/>Session &amp; Task Orchestration"]
        MEM["Memory &amp; Telemetry Service"]
        CORE <--> MEM
    end

    CORE -->|"spawn / stream"| CLI["Claude Code CLI"]
    CORE -->|"worktrees ¬∑ diffs"| GIT[("Git")]
    MEM -->|"vector search"| QDRANT[("Qdrant<br/>embeddings")]
    MEM -->|"metadata ¬∑ archive"| SQLITE[("SQLite<br/>sessions ¬∑ graph")]
    CLI -->|"MCP ¬∑ hooks"| MCP["MCP Servers"]
```

**Why a Swift + Rust split?** SwiftUI gives a truly native, low-latency Mac UI with system integration (menu bar, notifications, accessibility, voice). Rust gives a fast, memory-safe, single-binary core for concurrency-heavy orchestration and embeddings ‚ÄĒ no Node runtime, no Electron, no browser engine shipped inside the app.

## ūüöÄ Quickstart

### Prerequisites

- **macOS 14 (Sonoma) or newer**
- **[Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code)** installed and authenticated (`claude --version`)
- **Rust** (stable) via [rustup](https://rustup.rs) ‚ÄĒ `rustc --version`
- **Swift 6** toolchain (Xcode 16+ or the Swift 6 toolchain) ‚ÄĒ `swift --version`

### 1. Clone

```bash
git clone https://github.com/vqiz/ClaudeStudio.git
cd ClaudeStudio
```

### 2. Build the Rust core

```bash
cd core
cargo build --release
```

### 3. Run the core sidecar

```bash
cargo run --release
# The core listens on a local Unix domain socket and waits for the app to connect.
```

### 4. Build & run the macOS app

In a second terminal, from the repo root:

```bash
cd app
swift run ClaudeStudio
```

The app will connect to the running core, detect your Claude Code CLI, and open the workspace.

> First run? Open **Settings ‚Üí MCP & Hooks** to point ClaudeStudio at your existing MCP servers, and **Settings ‚Üí Trust** to pick a default permission mode.

## ūüóļÔłŹ Roadmap

ClaudeStudio is being built in four phases. Full detail lives in **[docs/roadmap.md](docs/roadmap.md)**.

| Phase | Theme | Highlights |
| ----- | ----- | ---------- |
| **1 ‚ÄĒ Foundation** | The native shell | Swift ‚áĄ Rust socket bridge, Claude CLI integration, session UI, permissions/trust modes |
| **2 ‚ÄĒ Memory** | Never start cold | Permanent session archive, Qdrant + SQLite semantic memory, Brain View knowledge graph |
| **3 ‚ÄĒ Agentic OS** | Agents that collaborate | Supervisor, Event-Bus, A2A messaging, Agent Studio & Teams |
| **4 ‚ÄĒ Ecosystem** | Power & polish | Voice Assistant, Definition & Task Libraries, MCP/Hooks managers, Cost & Telemetry |

> We are currently in **Phase 1**. Phases overlap and may reorder as we learn ‚ÄĒ follow the [milestones](https://github.com/vqiz/ClaudeStudio/milestones) for the live picture.
